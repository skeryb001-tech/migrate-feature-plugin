# 同项目跨页面迁移专项 Checklist

用于 `cross-page` 模式。迁移目标页面的同时必须保护源页面和共享消费者；先执行主 Skill 的共同流程，再把本清单并入对应阶段。

## 1. 页面上下文盘点

- [ ] 源页面、目标页面、路由入口和真实调用链已定位。
- [ ] 源页面当前行为与视觉基准已记录，迁移后仍需保留还是替换已明确。
- [ ] route params、query、hash、name、redirect 和动态路由 key 已对比。
- [ ] layout、middleware、page meta、权限守卫和 SEO 配置已对比。
- [ ] SSR/CSR、client-only、keep-alive、缓存页面和 hydration 边界已对比。
- [ ] 页面进入方式已覆盖直接 URL、站内导航、重定向和浏览器历史。
- [ ] 源页面私有组件/逻辑与项目共享能力已明确区分。
- [ ] 目标页面已有同类能力、同名文件、组件名、route name 和自动导入名已扫描。

## 2. 状态作用域与数据生命周期

- [ ] 每项状态已标记为组件私有、页面私有、会话共享或全局共享。
- [ ] 页面私有状态未为迁移方便而意外提升为全局 store。
- [ ] 需要共享的状态按业务 id、用户或会话正确隔离。
- [ ] 初始化、默认值、路由变化重置和再次进入恢复行为与源一致。
- [ ] keep-alive 下 activate/deactivate 行为已明确，不只验证 mount/unmount。
- [ ] 刷新后需要保留的数据使用目标已有持久化方式。
- [ ] 只持久化耐久、可序列化的源数据，不持久化 DOM、File、blob URL 或临时实例。
- [ ] A 页面与 B 页面同时打开时不会互相覆盖页面私有状态。

## 3. 共享能力与消费者保护

- [ ] 目标已有组件、composable、store、service、types、utils 和 token 已优先复用。
- [ ] 新增页面代码按目标项目页面、组件和业务模块结构落盘。
- [ ] 源结构只作为职责参考；最终目录、分层、命名以目标项目为准。
- [ ] 同名文件未覆盖；非等价能力已按目标规范语义化重命名。
- [ ] 修改共享组件/store/composable/service 前已扫描全部消费者。
- [ ] 已记录直接、间接、动态、自动导入和 barrel export 消费者。
- [ ] 共享契约默认向后兼容，源页面和目标既有页面无需隐式改写。
- [ ] 必须破坏共享契约时已升级 P0、说明影响并获得明确授权。
- [ ] 源页面、目标页面和所有关键消费者都有聚焦回归项。

## 4. 业务逻辑与副作用

- [ ] 输入、输出、默认值、校验、状态机和异常分支与源一致。
- [ ] 请求方法、参数、次数、顺序、重试、取消和错误反馈与源一致。
- [ ] 权限、登录、额度、支付、实验和埋点触发边缘与源一致。
- [ ] 请求、watcher、订阅、轮询、计时器和事件监听有唯一职责所有者。
- [ ] route params/query 变化不会重复初始化或漏刷新。
- [ ] 快速切页和乱序响应不会把旧页面结果写入当前页面。
- [ ] 页面离开、deactivate 或业务 id 变化时会取消/清理失效副作用。
- [ ] 返回页面时不会重复扣费、重复写入、重复埋点或重复弹窗。

## 5. 导航与页面序列

- [ ] A→B 的首次进入行为正确。
- [ ] B→A 返回后源页面行为、状态和 UI 未回归。
- [ ] A→B→A 连续导航无状态串页或重复副作用。
- [ ] 浏览器前进、后退和 history restore 行为正确。
- [ ] 直接刷新 A、直接刷新 B 均可恢复正确状态。
- [ ] 不同 params/query/hash 间切换行为正确。
- [ ] A、B 并行标签页或分屏使用时状态隔离正确。
- [ ] 登录态变化、权限变化和会话过期后的路由行为正确。

## 6. UI 与页面容器适配

- [ ] 目标页面中的逻辑、功能、交互和全部 UI 状态与源一致。
- [ ] 页面容器、header/footer/sidebar、安全区和滚动所有权已适配目标 layout。
- [ ] 弹层挂载点、z-index、焦点恢复和键盘操作在目标上下文可用。
- [ ] 相同数据、视口、缩放、语言、主题和字体下已完成源/目标视觉对照。
- [ ] 加载、空、成功、失败、禁用、hover、focus、选中和弹层状态完整。
- [ ] 响应式、长文案、极端数据、滚动恢复和 reduced-motion 已验证。
- [ ] UI 实现复用目标组件/token/i18n，未复制不符合目标规范的样式结构。

## 7. 清理、回滚与模式门禁

- [ ] 页面离开后 watcher、轮询、计时器、请求、observer 和事件监听均停止。
- [ ] 不再使用的页面私有状态、临时适配和旧调用点已清理。
- [ ] 源页面仍需保留时，其入口、功能、UI 和共享契约回归通过。
- [ ] 目标起始提交和新页面入口关闭方式已记录。
- [ ] 共享模块、API/埋点和持久化数据的恢复/清理方式已记录。
- [ ] 回滚目标页面不会破坏源页面或其他消费者。
- [ ] 本清单的结论和证据已写入迁移报告。

模式门禁：以上各项全部完成，源页面和目标既有消费者无回归，P0=0、P1=0，才允许判定 `cross-page` 专项通过。
