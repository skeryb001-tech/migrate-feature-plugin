# 跨项目迁移专项 Checklist

用于 `cross-project` 模式。先执行主 Skill 的共同流程，再把本清单并入对应阶段；任何 P0/P1 未通过时停止。

## 1. 资产与工程边界

- [ ] 源、目标仓库及相关 monorepo package 的生效规范已读取。
- [ ] 源功能真实调用链已追到 API、权限、状态、副作用和 UI 反馈。
- [ ] 两边的基准分支/提交、工作区状态和允许写入范围已记录。
- [ ] 框架、语言、渲染模式、构建工具和浏览器支持范围已对比。
- [ ] 包管理器、锁文件、workspace 配置和 package 边界已对比。
- [ ] 路径别名、自动导入、代码生成、环境变量和运行时配置已对比。
- [ ] 路由、layout、middleware、page meta、SSR/CSR 边界已对比。
- [ ] 目标目录规范、模块分层和命名规则已形成源职责到目标位置的映射。

## 2. 依赖与配置

- [ ] 运行包管理器命令前已读取目标 `package.json` 和 `packageManager`。
- [ ] 源依赖逐项映射为“目标已有依赖 / 目标封装 / 无需依赖 / 需授权”。
- [ ] 未把源锁文件、构建配置或整套基础设施复制到目标。
- [ ] 未经授权未新增、升级、降级或替换依赖。
- [ ] 环境变量只迁移名称和用途，不复制密钥、token 或生产值。
- [ ] 代理、base URL、CDN、资源 public path 和部署前缀已适配目标。
- [ ] 第三方 SDK、字体、图片和其他资产的许可与分发边界已确认。
- [ ] 服务端专用配置、客户端公开配置和 secret 边界没有混淆。

## 3. 目标能力复用与落盘

- [ ] 已按目标规定顺序搜索 demo、共享 UI、业务组件、hooks、stores、services、utils 和 types。
- [ ] 复用候选已逐项比较输入、输出、状态、时序、错误和视觉状态。
- [ ] 等价能力直接复用，没有迁入第二套实现。
- [ ] 部分等价能力通过最窄适配层收敛，没有在页面散落兼容逻辑。
- [ ] 新能力只迁入最小必要部分，并放入目标规范指定目录。
- [ ] 迁移后代码风格、目录、模块边界、文件命名和测试结构以目标项目为准。
- [ ] 源文件名仅在符合目标规则且无冲突时保留。
- [ ] 同名、大小写、路由、store id、导出和自动导入冲突均已扫描。
- [ ] 目标同名文件未被覆盖；非等价文件已语义化重命名并更新全部引用。

## 4. 数据、鉴权与业务契约

- [ ] 请求方法、路径、参数、header、序列化和响应解析保持业务语义一致。
- [ ] 目标请求层、错误格式、重试、超时、取消和鉴权机制已复用。
- [ ] 源字段到目标字段的适配集中在明确边界，组件不直接兼容多套协议。
- [ ] 权限、登录、额度、支付、实验和灰度分支与源行为一致。
- [ ] 缓存 key、持久化格式、失效策略和刷新恢复行为已对齐。
- [ ] 埋点事件名、触发时机、次数和 payload 语义已对齐。
- [ ] 未产生重复请求、重复写入、重复订阅或重复埋点。
- [ ] 跨 package 类型、导出和运行时依赖没有越过目标允许边界。

## 5. UI 与资源适配

- [ ] UI 状态和视觉结果以源功能为基准。
- [ ] 组件、设计 token、图标、i18n 和样式写法以目标项目为准。
- [ ] 所有目标支持语言已覆盖，未复制源项目不适用的 i18n 加载方式。
- [ ] 资源已进入目标规定的 assets/public/CDN 管线，构建后路径可用。
- [ ] 相同数据、视口、缩放、主题、语言和字体条件下已完成视觉对照。
- [ ] 桌面端、移动端、主题、长文案和极端数据状态已验证。
- [ ] 目标项目无同类全局样式、provider、plugin 或初始化逻辑被重复注册。

## 6. 消费者、构建与回归

- [ ] 修改共享能力前已扫描全部直接、间接、动态和自动导入消费者。
- [ ] 目标既有消费者的契约保持兼容，或破坏性变更已按 P0 获得授权。
- [ ] 目标 package 的 lint、typecheck、测试和构建已按项目方式执行。
- [ ] monorepo 受影响 package 和反向依赖已完成聚焦回归。
- [ ] 生产构建路径、按需加载、chunk、SSR 和 hydration 没有新增异常。
- [ ] 源功能主流程、关键分支、边界场景和 UI 状态在目标全部通过。
- [ ] 目标原有同类功能和共享消费者没有新增回归。

## 7. 回滚与模式门禁

- [ ] 目标起始提交已记录，可精确识别本次迁移改动。
- [ ] 新路由、入口、菜单或 feature flag 的关闭方法可执行。
- [ ] 公共组件、store、service、类型和导出的恢复方式已记录。
- [ ] API、埋点、环境配置和第三方初始化的恢复方式已记录。
- [ ] 新缓存、storage、IndexedDB 或持久化数据的清理方式已记录。
- [ ] 回滚不会删除目标原有数据、文件或用户改动。
- [ ] 本清单的结论和证据已写入迁移报告。

模式门禁：以上各项全部完成，P0=0、P1=0，才允许判定 `cross-project` 专项通过。
